- 2026-05-31T15:56:07Z [sarocu] PLAN (content index; lives in the AGENT, which has /data filesystem access)

Scope: full-text index over ALL user data so the agent can search it. Real tree-sitter is NOT used here — the data is prose/JSON, not source code; we do markdown heading-aware sectioning instead and note that explicitly.

New file: agent/src/tools/search.ts
- Doc model: IndexDoc { kind: todo|backlog|spec|list|list_item|book|page|artifact; id; date?; list_id?; item_id?; book_id?; page_id?; title; body; heading?; updated_at; token } where `token` is the agent-readable mention string used by chat chips (sets up T-016/T-017).
- Extractors (reuse existing loaders, do NOT re-implement storage):
  - todos+specs: recurse /data/storage/YYYY/MM/DD/day.json (fs.readdir walk, skip lists/ books/ backlog/ subtrees); per todo: title + loadSpecContent(date,id); token @[title](id:<id>|date:<date>).
  - backlog: loadBacklog() items + loadBacklogSpec(id); token @[title](id:<id>|date:) (empty date).
  - lists: listLists()->getList(id); one doc for the list + one per item (body = JSON of values); tokens @[name](list:<id>) and @[firstvalue](list:<id>/<itemId>).
  - books: listBooks()->getBook(id); doc for book + per-page doc (title + loadPage md); tokens @[name](book:<id>) and @[pageTitle](book:<id>/<pageId>).
  - artifacts: read /data/artifacts/.index.json; read body for text-ish files (skip binary by ext); token @[name](artifact:<relpath>).
- Markdown sectioning ("codegraph style"): split md bodies on ^#{1,6} headings into heading-scoped fragments; index each fragment with its nearest heading so results can show a section subtitle. Plain regex, no new dependency (agent has no markdown parser dep).
- Index: tokenize lowercase split on non-alphanumeric -> inverted index Map<term, Set<docId>> + doc store. Rank by term-frequency with a title-match boost; substring fallback for short queries. Build snippet around first hit.
- Freshness: persist to /data/index/index.json with a per-source-file mtime map (mirrors the existing .codegraph/index.json mtime pattern); buildIndex(force?) stats each source and only re-extracts changed files -> lazy rebuild at query time, so NO write-path hooks are needed. (Write-time invalidation hooks are a possible future optimization, out of scope.)
- Exports: buildIndex(force?) and searchIndex(query, { kinds?, limit? }): IndexDoc[] (ranked, with snippet).
