## Notes

- 2026-05-31T15:56:31Z [sarocu] PLAN (no search bar — a built-in "/search" skill in the chat input, Claude-skills style)

Depends on T-015 (the search_content tool).

Concept: a leading slash command in chat = a built-in skill. Start with just /search. Reuse the existing @-mention menu machinery in ChatInput.tsx rather than building a separate widget.

src/skills.ts (new): export const SKILLS = [{ name: "search", hint: "Search across todos, lists, books, and artifacts" }] so the menu is data-driven/extensible.

src/components/chat/ChatInput.tsx:
- In handleInput, alongside the existing @-trigger regex, detect when the textarea value starts with "/" and the caret is still in the first token (no space yet) -> setShowSkills(true) + skillQuery = text after the slash; filter SKILLS by prefix.
- Render a skill menu reusing `.mention-menu`/`.mention-item` classes (new state showSkills/skillIndex, parallel to showMentions). ArrowUp/Down/Enter/Tab to select; selecting autocompletes the textarea to "/search " (a literal prefix — do NOT expand into a hidden token the way mentions do).
- handleKeyDown: when the skill menu is open, intercept arrows/Enter/Tab/Escape (mirror the existing showMentions branch).
- On submit: send the message verbatim, including the leading "/search <query>". The agent recognizes the convention (see prompt change below) and routes to search_content. This keeps it a zero-new-RPC "skill".

agent/src/agent.ts system prompt: add that a user message beginning with "/search " means: treat the remainder as a query, call search_content, and present results as the clickable `token` chips. (Pairs with the T-015 prompt note.)

Optional/nice-to-have (note, not required): render a leading /search as a small skill chip in the user bubble (MentionText/MessageBubble).

Verify:
- npm run build (root: tsc && vite) clean.
- Manual: type "/", menu shows /search; pick it, type a query, send; agent returns chip results. Confirm @-mentions still work (the two menus must not fight — slash only triggers at start-of-message, @ anywhere).
